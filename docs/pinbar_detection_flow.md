# Pinbar 信号检测完整流程

## 一、核心检测流程图

```
┌─────────────────────────────────────────────────────────────────┐
│                    新 K 线闭合事件 (WebSocket)                    │
└─────────────────────────┬───────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────────┐
│  前置检查：K 线是否已闭合？数据是否足够 (≥60 根用于 EMA60)?          │
└─────────────────────────┬───────────────────────────────────────┘
                          │
          ┌──────────────┴──────────────┐
          │ Yes                         │ No
          ▼                             ▼
┌─────────────────────────┐   ┌─────────────────┐
│  计算核心指标            │   │  跳过，仅缓存 K 线  │
│  • EMA60 = calculate_ema │   └─────────────────┘
│  • ATR14 = calculate_atr │
└───────────┬─────────────┘
            │
            ▼
┌─────────────────────────────────────────────────────────────────┐
│                    第一层过滤：形态识别 (Shape Filter)            │
│  ┌───────────────────────────────────────────────────────────┐  │
│  │ 1. 实体过滤：body_length <= total_length * body_max_ratio │  │
│  │    (默认：实体 ≤ 25% 总长)                                   │  │
│  │                                                           │  │
│  │ 2. Doji 边界处理：                                         │  │
│  │    - 若 body_ratio < 5% → 视为十字星                         │  │

│  │                                                           │  │
│  │ 3. 影线验证：                                              │  │
│  │    - 下影线 >= body * effective_shadow_ratio → 做多信号    │  │
│  │    - 上影线 >= body * effective_shadow_ratio → 做空信号    │  │
│  │    - 双向满足取更长一方；都不满足 → 拒绝                    │  │
│  └───────────────────────────────────────────────────────────┘  │
└─────────────────────────┬───────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────────┐
│                    第二层过滤：趋势背离 (Trend Filter)           │
│  ┌───────────────────────────────────────────────────────────┐  │
│  │ EMA60 趋势判断：                                            │  │
│  │  • close > EMA60 → 多头趋势                                 │  │
│  │  • close < EMA60 → 空头趋势                                 │  │
│  │                                                           │  │
│  │ MTF 大级别趋势过滤 (可选):                                   │  │
│  │  • Hard 模式：逆势信号直接拒绝                              │  │
│  │  • Soft 模式：允许逆势，但评分 -15 分                          │  │
│  │                                                           │  │
│  │ 形态背离标记：                                             │  │
│  │  • 做多信号但 close < EMA60 → is_shape_divergent = True   │  │
│  │  • 做空信号但 close > EMA60 → is_shape_divergent = True   │  │
│  └───────────────────────────────────────────────────────────┘  │
└─────────────────────────┬───────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────────┐
│                    第三层过滤：波动率过滤 (Volatility Filter)    │
│  ┌───────────────────────────────────────────────────────────┐  │
│  │ 活跃度验证：total_length > volatility_atr_multiplier * ATR │  │
│  │  (默认：K 线总长 > 1.2 × ATR14)                              │  │
│  │  避免在死水/震荡期识别无意义的小级别 Pinbar                   │  │
│  └───────────────────────────────────────────────────────────┘  │
└─────────────────────────┬───────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────────┐

│  ┌───────────────────────────────────────────────────────────┐  │
│  │ 1. 计算止损距离：                                          │  │
│  │    • 做多：sl_distance = (close - low) / close             │  │
│  │    • 做空：sl_distance = (high - close) / close            │  │
│  │                                                           │  │
│  │ 2. 动态止损阈值 (ATR 自适应):                               │  │
│  │    dynamic_sl = base + (ATR/price) × multiplier           │  │

│  │                                                           │  │
│  │ 3. 天地针熔断：sl_distance_pct > effective_max_sl_dist → 拒绝 │  │
│  └───────────────────────────────────────────────────────────┘  │
└─────────────────────────┬───────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────────┐
│                    止盈计算 (1.5R 盈亏比)                          │
│  • TP1 = entry_price ± (stop_loss_distance × 1.5)               │
└─────────────────────────┬───────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────────┐
│                    动态评分计算 (Dynamic Scoring)                │
│  ┌───────────────────────────────────────────────────────────┐  │
│  │ S_shape (形态完美度 0-100):                                 │  │
│  │   - 基于 shadow_ratio，0.6→0 分，0.9+→100 分                  │  │
│  │                                                           │  │
│  │ S_trend (趋势回归分 0-100):                                 │  │
│  │   - 基于距离 EMA60 的远近，0.5% 内→高分，>3%→0 分               │  │
│  │                                                           │  │
│  │ S_vol (波动爆发分 0-100):                                   │  │
│  │   - 基于 ATR 倍数，1.2x→0 分，3.0x→100 分                       │  │
│  │                                                           │  │
│  │ 最终得分 = S_shape×w_shape + S_trend×w_trend + S_vol×w_vol  │  │
│  │                                                           │  │
│  │ 额外扣分项：                                               │  │
│  │   - 形态背离：-20 分                                         │  │
│  │   - MTF 逆势 (soft 模式): -15 分                               │  │
│  └───────────────────────────────────────────────────────────┘  │
└─────────────────────────┬───────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────────┐
│                    生成 Signal 对象并返回                        │
│  • 附带全部元数据：shadow_ratio, ema_distance, volatility_atr  │
│  • is_contrarian, is_shape_divergent 标记                      │
└─────────────────────────────────────────────────────────────────┘
```

## 二、影响参数完整清单

### 1. Pinbar 形态识别参数 (`PinbarConfig`)

| 参数 | 默认值 | 说明 | 影响阶段 |
|------|--------|------|----------|
| `body_max_ratio` | 0.25 | 实体最大占总长比例 | 形态过滤 |
| `shadow_min_ratio` | 2.5 | 影线最小为实体的倍数 | 形态过滤 |
| `volatility_atr_multiplier` | 1.2 | K 线总长相对 ATR 的最小倍数 | 波动率过滤 |
| `doji_threshold` | 0.05 | 十字星判定阈值 (实体/全长<5%) | Doji 边界 |
| `doji_shadow_bonus` | 0.6 | 十字星影线比例放宽系数 (2.5×0.6=1.5) | Doji 边界 |
| `mtf_trend_filter_mode` | "soft" | "soft"(降分) 或 "hard"(直接拒绝) | MTF 过滤 |
| `dynamic_sl_enabled` | true | 是否启用动态止损阈值 | 止损风控 |
| `dynamic_sl_base` | 0.035 | 动态止损基准值 (3.5%) | 止损风控 |
| `dynamic_sl_atr_multiplier` | 0.5 | ATR 对止损的贡献系数 | 止损风控 |
| `shape_divergence_penalty` | 20 | 形态 - 趋势背离扣分数 | 评分计算 |

### 2. 评分权重参数 (`ScoringWeights`)

| 参数 | 范围 | 说明 | 影响 |
|------|------|------|------|
| `w_shape` | 0-1 | 形态完美度权重 | 影响总分中 Shape 分的占比 |
| `w_trend` | 0-1 | 趋势回归度权重 | 影响总分中 Trend 分的占比 |
| `w_vol` | 0-1 | 波动率权重 | 影响总分中 Vol 分的占比 |

**约束**: `w_shape + w_trend + w_vol = 1.0`

### 3. 风控参数 (`RiskConfig`)

| 参数 | 默认值 | 说明 | 影响阶段 |
|------|--------|------|----------|
| `risk_pct` | 0.02 | 单笔风险占总资金比例 (2%) | 算仓 |
| `max_sl_dist` | 0.035 | 最大止损距离 (3.5%) | 止损风控 |
| `max_leverage` | 20.0 | 最大允许杠杆倍数 | 算仓 |

### 4. 监控配置 (`IntervalConfig`)

| 参数 | 说明 | 影响 |
|------|------|------|
| `use_trend_filter` | 是否开启大级别趋势校验 | MTF 趋势过滤 |

**MTF 映射关系**:
- 15m → 参考 1h 趋势
- 1h → 参考 4h 趋势
- 4h → 参考 1d 趋势
- 1d → 参考自身 (无更大级别)

## 三、关键计算公式

### 1. 实体与影线计算
```python
body_length = abs(open - close)
total_length = high - low
lower_shadow = min(open, close) - low
upper_shadow = high - max(open, close)
```

### 2. Doji 判定与影线放宽
```python
body_ratio = body_length / total_length
is_doji = body_ratio < 0.05  # doji_threshold
effective_shadow_ratio = shadow_min_ratio * (0.6 if is_doji else 1.0)
```

### 3. 动态止损阈值
```python
atr_volatility = atr14 / entry_price
dynamic_sl = dynamic_sl_base + (atr_volatility * dynamic_sl_atr_multiplier)
effective_max_sl_dist = min(dynamic_sl, max_sl_dist * 1.5)
```

### 4. 评分计算
```python
# S_shape: 0-100 (0.6→0 分，0.9→100 分)
s_shape = (shadow_ratio - 0.6) / 0.3 * 100

# S_trend: 0-100 (0.5% 内→高分，>3%→0 分)
s_trend = (0.03 - dist_to_ema) / 0.025 * 100

# S_vol: 0-100 (1.2x→0 分，3.0x→100 分)
s_vol = (vol_ratio - 1.2) / 1.8 * 100

# 最终得分
score = s_shape * w_shape + s_trend * w_trend + s_vol * w_vol
score -= (20 if is_shape_divergent else 0)
score -= (15 if mtf_contrarian else 0)
```

## 四、当前检测结构的问题诊断

根据代码分析，可能的不满之处：

### 1. 级联过滤过于严格
四层过滤串行执行，任何一层失败都直接返回 `None`，可能导致大量"准 Pinbar"被漏掉。

### 2. 评分与过滤耦合
评分逻辑在过滤之后执行，但部分评分因子 (如 `ema_distance`) 本可以用于更精细的过滤。

### 3. MTF 趋势计算时机
MTF 高级别 EMA60 在每次 K 线闭合时重新计算，未做缓存优化。

### 4. 信号方向二选一
当上下影线都满足条件时，简单取更长一方，未考虑"双向 Pinbar"的特殊形态。

### 5. 固定阈值过滤
所有参数都是静态的，未根据市场状态 (如横盘/趋势) 动态调整。
