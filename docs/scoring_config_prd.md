# 动态可配置打分机制 - 产品需求文档 (PRD)

**版本**: v1.0
**日期**: 2026-03-03
**状态**: 待开发

---

## 1. 文档概述

### 1.1 需求背景

当前 Pinbar 信号打分系统采用**硬编码的线性评分公式**，存在以下问题：

1. **参数固定** - 影线比例、波动率等阈值硬编码在代码中，无法根据市场状态调整
2. **模式单一** - 只有经典线性评分一种模式，边缘信号难以获得合理分数
3. **缺乏灵活性** - 用户无法自定义打分行为，无法针对不同币种/周期定制策略

### 1.2 需求目标

构建一套**动态可配置的打分系统**，实现：

1. ✅ 支持多种打分模式（经典/累进/自定义）
2. ✅ 所有评分参数可通过 API 热配置
3. ✅ 配置持久化到数据库，重启后保留
4. ✅ 前端提供可视化配置界面和分数预览

### 1.3 名词解释

| 术语 | 解释 |
|------|------|
| Pinbar | Pinbar 形态，实体小、影线长的 K 线形态 |
| MTF | Multi-TimeFrame，多重时间框架分析 |
| ATR | Average True Range，平均真实波幅 |
| EMA60 | 60 周期指数移动平均线 |
| 经典模式 | 原有的线性评分模式 (0.6→0 分，0.9→100 分) |
| 累进模式 | 新增的阶梯加分模式 (基础分 + 奖励分) |
| 自定义模式 | 用户可自定义公式系数的模式 |

## 2. 用户故事

### 2.1 核心用户故事

| ID | 作为 | 我想要 | 以便于 |
|----|------|--------|--------|
| US1 | 交易员 | 切换不同的打分模式 | 适应不同市场状态（震荡/趋势） |
| US2 | 交易员 | 调整影线比例阈值 | 更严格/宽松地筛选 Pinbar 形态 |
| US3 | 交易员 | 配置波动率奖励参数 | 鼓励/抑制高波动信号 |
| US4 | 管理员 | 保存配置并持久化 | 系统重启后配置不丢失 |
| US5 | 交易员 | 预览分数分布 | 了解配置调整后的实际效果 |

### 2.2 使用场景

#### 场景 1：震荡市配置
```
市场状态：横盘震荡，假突破多
用户操作：
  - 切换到"累进模式"
  - 提高影线奖励阈值 (0.6 → 0.7)
  - 降低实体奖励阈值 (0.1 → 0.08)
预期效果：过滤掉更多边缘信号，只保留高质量 Pinbar
```

#### 场景 2：趋势市配置
```
市场状态：单边趋势，回调入场机会多
用户操作：
  - 切换到"经典模式"或"累进模式"
  - 降低波动率奖励起点 (2.0x → 1.5x)
  - 提高趋势穿透奖励
预期效果：捕捉更多顺势回调信号
```

#### 场景 3：小币种配置
```
监控标的：波动大的小市值币种
用户操作：
  - 切换到"自定义模式"
  - 降低形态权重 (0.4 → 0.3)
  - 提高波动权重 (0.3 → 0.4)
预期效果：适应小币种高波动特性
```

## 3. 功能需求

### 3.1 打分模式切换

**F1.1** 系统必须支持三种打分模式：
- `classic` - 经典线性模式（保持现有行为）
- `progressive` - 累进加分模式（新增）
- `custom` - 自定义公式模式（扩展）

**F1.2** 模式切换必须实时生效，无需重启服务

**F1.3** 模式切换后，历史信号分数不变，仅影响新产生信号

---

### 3.2 经典模式参数配置

**F2.1** 用户可配置以下参数：

| 参数 | 说明 | 默认值 | 可调范围 |
|------|------|--------|----------|
| `classic_shadow_min` | 影线比例最小值 (0 分) | 0.6 | 0.3 - 0.8 |
| `classic_shadow_max` | 影线比例最大值 (100 分) | 0.9 | 0.7 - 1.0 |
| `classic_body_good` | 实体比例优秀阈值 | 0.1 | 0.01 - 0.3 |
| `classic_body_bad` | 实体比例差阈值 | 0.5 | 0.3 - 0.7 |
| `classic_vol_min` | 波动率最小值 (0 分) | 1.2x | 0.8 - 2.0x |
| `classic_vol_max` | 波动率最大值 (100 分) | 3.0x | 2.0 - 5.0x |
| `classic_trend_max_dist` | 趋势距离最大值 (0 分) | 0.03 | 0.01 - 0.1 |

**F2.2** 参数调整后，新产生的信号立即使用新参数计算

---

### 3.3 累进模式参数配置

**F3.1** 累进模式采用**基础分 + 奖励分**机制

**F3.2** 用户可配置以下参数：

| 参数 | 说明 | 默认值 | 可调范围 |
|------|------|--------|----------|
| `progressive_base_cap` | 每个维度基础分上限 | 30.0 | 20 - 50 |
| `progressive_shadow_threshold` | 影线奖励阈值 | 0.6 | 0.4 - 0.8 |
| `progressive_shadow_bonus_rate` | 影线奖励倍率 (分/0.1) | 20.0 | 10 - 50 |
| `progressive_body_bonus_threshold` | 实体奖励阈值 | 0.1 | 0.05 - 0.2 |
| `progressive_body_bonus_rate` | 实体奖励倍率 (分/0.01) | 100.0 | 50 - 200 |
| `progressive_doji_bonus` | 十字星固定奖励分 | 5.0 | 0 - 20 |
| `progressive_vol_threshold` | 波动率奖励起点 | 2.0x | 1.2 - 3.0x |
| `progressive_vol_bonus_rate` | 波动率奖励倍率 (分/0.1x) | 15.0 | 5 - 30 |
| `progressive_penetration_rate` | 趋势穿透奖励倍率 | 30.0 | 10 - 50 |

**F3.3** 累进模式计算公式详见技术文档

---

### 3.4 权重配置

**F4.1** 用户可配置三个维度的权重：

| 权重 | 说明 | 默认值 | 约束 |
|------|------|--------|------|
| `w_shape` | 形态评分权重 | 0.4 | w_shape + w_trend + w_vol = 1.0 |
| `w_trend` | 趋势评分权重 | 0.3 | - |
| `w_vol` | 波动评分权重 | 0.3 | - |

**F4.2** 权重总和必须严格等于 1.0 (100%)，否则拒绝保存

---

### 3.5 配置持久化

**F5.1** 所有配置参数必须持久化到 SQLite 数据库

**F5.2** 服务重启后，自动加载上次保存的配置

**F5.3** 配置变更时，记录操作日志（可选）

---

### 3.6 分数预览功能

**F6.1** 前端提供分数分布预览图表

**F6.2** 用户调整参数时，实时更新预览图表

**F6.3** 预览图表展示：
- X 轴：分数区间 (0-20, 20-40, ..., 80-100)
- Y 轴：信号数量（基于最近 N 条历史信号重算）

## 4. 打分模式对比

### 4.1 经典模式 (Classic)

```
评分逻辑：线性映射
适用场景：市场稳定，信号质量均匀分布

形态分计算:
  shadow_score = (shadow_ratio - 0.6) / 0.3 * 100  # 0.6→0, 0.9→100
  body_score = (0.5 - body_ratio) / 0.4 * 100      # 0.1→100, 0.5→0
  shape_score = shadow_score * 0.4 + body_score * 0.3 + ratio_score * 0.3
```

### 4.2 累进模式 (Progressive)

```
评分逻辑：基础分 + 奖励分
适用场景：需要区分信号质量，精品信号高亮

形态分计算:
  base_shape = min(30, 基础分)
  bonus_shape = (shadow_ratio - 0.6) * 200  # 每 +0.1 奖励 2 分
              + (0.1 - body_ratio) * 100    # 每 -0.01 奖励 1 分
              + 5 (如果是十字星)
              + (shadow_to_body - 3) * 2    # 影线/实体>3 后每 +1 奖励 2 分
  shape_score = base_shape + min(40, bonus_shape)
```

### 4.3 预期分数对比

| 信号类型 | 经典模式 | 累进模式 | 差异 |
|---------|---------|---------|------|
| 边缘 Pinbar (影线 65%, 实体 20%) | 40-50 分 | 45-55 分 | +5 分 |
| 普通 Pinbar (影线 75%, 实体 15%) | 55-65 分 | 60-70 分 | +5 分 |
| 精品 Pinbar (影线 85%, 实体 8%) | 70-80 分 | 75-90 分 | +10 分 |
| 极端 Pinbar (影线 92%, 实体 3%) | 85-95 分 | 90-100 分 | +5 分 |

## 5. 非功能需求

### 5.1 性能要求

- **NFR1**: 配置接口响应时间 < 100ms
- **NFR2**: 打分计算额外开销 < 1ms（相比硬编码）
- **NFR3**: 配置热加载不影响正在处理的信号

### 5.2 可靠性要求

- **NFR4**: 配置异常时（如参数非法），保持使用上一次有效配置
- **NFR5**: 数据库故障时，使用内存中的默认配置继续运行

### 5.3 可用性要求

- **NFR6**: 前端提供配置重置按钮，一键恢复默认值
- **NFR7**: 关键参数提供"推荐值"提示
