### 1. 后端信号检测流程标准化 (Backend Logic)

为了确保信号检测的严谨性与透明度，后端需要建立标准化的处理全流程：

- **数据抓取层**：根据配置的时间级别（15m, 1h, 4h, 1d），异步并发抓取币安 K 线数据。
- **形态识别层**：执行 Pinbar 数学判定公式，计算影线占比与实体比例。
- **趋势过滤层**：引入大级别趋势校验逻辑，判定当前信号是否“顺势”。
- **动态评分层**：结合用户设置的 Shape、Trend、Vol 权重，生成最终综合评分。
- **持久化与推送**：将合格信号存入 SQLite 并触发飞书/企微多通道推送。

### 2. 多时间级别 (Interval) 监测与展示

系统需支持对不同时间维度的信号进行独立监测与清晰展示。

- **数据标识**：在信号列表中新增“级别”字段，明确展示信号产生的时间周期（如 15min, 1h, 4h, 1d）。
- **列表集成**：在“币种”列旁或通过独立列展示该级别，确保用户一眼能分辨不同周期的机会。

### 3. 多重时间框架 (MTF) 趋势参考逻辑

这是 5.0 版本最核心的算法升级，通过“小级别服从大级别”的逻辑减少噪音：

- **参考映射关系**：
  - **15min 信号**：必须参考 **1h** 级别的 EMA60 趋势方向。
  - **1h 信号**：必须参考 **4h** 级别的 EMA60 趋势方向。
  - **4h 信号**：必须参考 **1d** 级别的 EMA60 趋势方向。
- **逻辑公式**：
  - 仅当 $Signal_{side} == Trend_{higher\_interval}$ 时，信号被判定为有效。
  - 例如：1h EMA60 向上（多头趋势），则仅记录 15min 的做多（Long）Pinbar。

### 4. 动态列表表头配置 (Dynamic View Config)

提升 UI 的自定义程度，让表格完全适配用户的视觉偏好。

- **交互功能**：在信号列表右上方增加“列显示配置”按钮。
- **功能定义**：
  - 提供勾选清单：发生时间、级别、币种、方向、综合评分、入场价、影线占比、EMA 距离、ATR 波动率等。
  - 用户勾选则显示，取消勾选则隐藏。
- **持久化存储**：使用浏览器 `localStorage` 记住用户的配置，刷新后无需重设。

### 5. 系统设置页功能增强 (Settings Upgrade)

- **监控级别多选**：在配置页增加“监控周期”多选框，选项包含：`15m`, `1h`, `4h`, `1d`。
- **动态配置交互**：用户勾选相应周期后，后端自动开启对应的 MTF 趋势监测任务。
- **安全开关维持**：继续保持“自动下单”开关为置灰且不可编辑状态，确保系统安全性。
