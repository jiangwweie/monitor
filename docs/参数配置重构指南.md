## 1. 重构背景

### 1.1 原有问题

重构前系统配置管理存在以下问题：

| 问题 | 描述 | 影响 |
|------|------|------|
| 配置刷新问题 | 修改配置后刷新页面显示旧配置 | 用户体验差 |
| 保存逻辑不统一 | 有的配置即时生效，有的需要点保存 | 用户困惑 |
| 配置未分类 | 所有配置挤在一个页面 | 难以查找和管理 |
| 无导入导出 | 无法备份和迁移配置 | 运维不便 |

### 1.2 原有架构

```
前端：Settings.tsx (单一组件，所有配置)
后端：PUT /api/config (统一接口，混合所有配置)
```

---

## 2. 重构目标

### 2.1 功能目标

1. **配置可编辑**：所有参数可在前端编辑
2. **后端入库**：配置持久化到 SQLite
3. **动态刷新**：配置修改后无需重启引擎
4. **统一保存**：所有配置修改需点击保存
5. **按类拆分**：设置页按功能分为子 Tab
6. **导入导出**：支持配置备份和恢复

### 2.2 技术目标

1. 前后端接口按配置类型拆分
2. 完善的配置校验机制
3. 敏感信息保护（API Key 导出置空）
4. 类型安全的 Pydantic 模型验证

#### ConfigService

```python
class ConfigService:
    """配置服务类，统一管理配置的读取和更新"""

    def __init__(self, repo):
        self.repo = repo  # SQLiteRepo 实例

    # 获取配置
    async def get_exchange_config() -> Dict
    async def get_monitor_config() -> Dict
    async def get_pinbar_config() -> Dict
    async def get_risk_config() -> Dict
    async def get_scoring_config() -> Dict
    async def get_push_config() -> Dict

    # 更新配置
    async def update_exchange_config(config: Dict) -> Dict
    async def update_monitor_config(config: Dict) -> Dict
    async def update_pinbar_config(config: Dict) -> Dict
    async def update_risk_config(config: Dict) -> Dict
    async def update_scoring_config(config: Dict) -> Dict
    async def update_push_config(config: Dict) -> Dict

    # 导入导出
    async def get_all_config_for_export() -> Dict
    async def import_config_from_yaml(config: Dict, engine) -> Dict
```

### 4.3 接口实现示例

```python
@app.put("/api/config/push")
async def update_push_config(request: Request, req: PushConfigReq):
    """
    更新推送配置
    支持热更新各推送通道的启用状态和密钥
    """
    repo = request.app.state.repo
    config_service = ConfigService(repo)

    update_data = req.model_dump(exclude_unset=True)

    try:
        data = await config_service.update_push_config(update_data)
        return _create_response(data, message="推送配置已热更新")
    except Exception as e:
        logger.error(f"推送配置更新失败：{e}")
        raise HTTPException(status_code=500, detail=str(e))
```

```tsx
// 以 PinbarTab 为例
export function PinbarTab({ initialConfig, onConfigChange }) {
  const [loading, setLoading] = useState(false);
  const [formData, setFormData] = useState({
    body_max_ratio: 0.25,
    shadow_min_ratio: 2.5,
    // ...
  });

  // 同步初始配置
  useEffect(() => {
    setFormData({
      body_max_ratio: initialConfig.body_max_ratio ?? 0.25,
      shadow_min_ratio: initialConfig.shadow_min_ratio ?? 2.5,
      // ...
    });
  }, [initialConfig]);

  // 保存处理
  const handleSave = async () => {
    setLoading(true);
    try {
      const res = await fetch("http://localhost:8000/api/config/pinbar", {
        method: "PUT",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify(formData),
      });

      if (res.ok) {
        toast.success("Pinbar 策略配置已保存");
        // 更新父组件 state
        Object.entries(formData).forEach(([key, value]) => {
          onConfigChange(key, value);
        });
      }
    } catch (error) {
      toast.error("保存失败");
    } finally {
      setLoading(false);
    }
  };

  return (
    <Card>
      <CardContent>
        {/* 表单控件 */}
        <Button onClick={handleSave} disabled={loading}>
          {loading ? "保存中..." : "保存 Pinbar 策略配置"}
        </Button>
      </CardContent>
    </Card>
  );
}
```

## 6. 配置校验规则

### 6.1 异常类定义

```python
# domain/exceptions.py
class ConfigValidationError(Exception):
    """配置校验失败异常"""
    pass
```

### 6.2 校验规则

| 配置项 | 校验规则 | 错误消息 |
|--------|---------|---------|
| monitor_intervals | 至少保留一个级别 | "至少需要保留一个监控级别" |
| w_shape + w_trend + w_vol | 总和必须为 1.0 | "打分权重总和必须为 1.0" |
| risk_pct | 0.005 - 0.1 | "risk_pct 必须在 0.005-0.1 范围内" |
| max_sl_dist | 0.01 - 0.1 | "max_sl_dist 必须在 0.01-0.1 范围内" |
| max_leverage | 1 - 125 | "max_leverage 必须在 1-125 范围内" |
| body_max_ratio | 0.05 - 0.8 | "body_max_ratio 必须在 0.05-0.8 范围内" |
| shadow_min_ratio | 1.0 - 10.0 | "shadow_min_ratio 必须在 1.0-10.0 范围内" |
| volatility_atr_multiplier | 0.5 - 5.0 | "volatility_atr_multiplier 必须在 0.5-5.0 范围内" |

### 6.3 校验实现示例

```python
async def import_config_from_yaml(self, config: Dict[str, Any], engine: Any = None) -> Dict[str, Any]:
    """从 YAML 配置字典导入配置，包含完整的校验逻辑"""

    # 1. 校验 monitor_intervals 至少保留一个级别
    if "monitor_config" in config or "monitor_intervals" in config:
        monitor_intervals = config.get("monitor_config", config.get("monitor_intervals", {}))
        if isinstance(monitor_intervals, dict) and len(monitor_intervals) == 0:
            raise ConfigValidationError("监控配置：至少需要保留一个监控级别")

    # 2. 校验打分权重总和为 1.0
    if "scoring_weights" in config:
        w = config["scoring_weights"]
        w_shape = w.get("w_shape", 0)
        w_trend = w.get("w_trend", 0)
        w_vol = w.get("w_vol", 0)
        total = round(float(w_shape) + float(w_trend) + float(w_vol), 4)
        if abs(total - 1.0) > 0.0001:
            raise ConfigValidationError(
                f"打分权重总和必须为 1.0，当前总和为 {total}"
            )

    # 3. 校验风险参数范围
    if "risk_config" in config:
        risk = config["risk_config"]
        risk_pct = float(risk.get("risk_pct", 0))
        if not (0.005 <= risk_pct <= 0.1):
            raise ConfigValidationError(
                f"风险配置：risk_pct 必须在 0.005-0.1 范围内，当前为 {risk_pct}"
            )
        # ... 其他参数校验

    # 4. 校验通过，开始导入
    # ...
```

## 7. 导入导出功能

### 7.1 导出功能

**接口**: `POST /api/config/export`

**实现逻辑**:

```python
@app.post("/api/config/export")
async def export_config(request: Request):
    """导出配置为 YAML 文件，敏感字段置空"""
    repo = request.app.state.repo
    config_service = ConfigService(repo)

    # 获取所有配置
    all_config = await config_service.get_all_config_for_export()

    # 生成带时间戳的文件名
    timestamp = datetime.now().strftime("%Y%m%d_%H%M%S")
    export_dir = "config"
    os.makedirs(export_dir, exist_ok=True)
    file_path = os.path.join(export_dir, f"exported_config_{timestamp}.yaml")

    # 写入 YAML 文件
    with open(file_path, "w", encoding="utf-8") as f:
        yaml.dump(all_config, f, allow_unicode=True, default_flow_style=False)

    return _create_response({
        "file_path": file_path,
    }, message=f"配置已导出到 {file_path}")
```

### 7.3 导入功能

**接口**: `POST /api/config/import`

**请求格式**: `multipart/form-data`

| 字段 | 类型 | 说明 |
|------|------|------|
| file | UploadFile | YAML 文件（可选） |
| yaml_content | Form | YAML 内容字符串（可选） |

**实现逻辑**:

```python
@app.post("/api/config/import")
async def import_config(
    request: Request,
    file: Optional[UploadFile] = File(None),
    yaml_content: Optional[str] = Form(None)
):
    """从 YAML 导入配置，支持文件上传或直接传入 yaml_content"""
    repo = request.app.state.repo
    engine = request.app.state.engine
    config_service = ConfigService(repo)

    content = yaml_content

    # 如果上传了文件，读取文件内容
    if file:
        if not file.filename.endswith((".yaml", ".yml")):
            raise HTTPException(status_code=400, detail="文件格式错误")
        file_bytes = await file.read()
        content = file_bytes.decode("utf-8")

    if not content:
        raise HTTPException(status_code=400, detail="yaml_content 或 file 必须提供一个")

    # 解析 YAML
    config = yaml.safe_load(content)

    # 校验并导入配置
    imported_config = await config_service.import_config_from_yaml(config, engine)

    return _create_response({
        "imported_config": imported_config,
    }, message="配置导入成功")
```

## 8. 注意事项

### 8.1 安全注意事项

| 注意点 | 说明 |
|--------|------|
| API Key 导出置空 | 导出的 YAML 文件中 `binance_api_key` 和 `binance_api_secret` 必须置空 |
| API Key 导入跳过 | 导入时不覆盖 API Key，需手动配置 |
| 只读权限 | API Key 应仅开启只读权限，严禁开启交易权限 |
| 加密存储 | 敏感字段在 SQLite 中应加密存储 |

### 8.2 技术注意事项

| 注意点 | 说明 |
|--------|------|
| 热更新顺序 | 先更新 Engine 内存，再写入 SQLite |
| 配置校验时机 | 在写入前校验，失败则回滚 |
| 并发控制 | 多个 Tab 同时保存时注意数据一致性 |
| 错误处理 | 捕获 `ConfigValidationError` 并返回友好的错误消息 |

### 8.3 前端注意事项

| 注意点 | 说明 |
|--------|------|
| 配置刷新 | 保存成功后建议重新拉取配置或刷新页面 |
| Toast 提示 | 保存成功/失败都要有明确的提示 |
| Loading 状态 | 保存期间按钮应显示 loading 状态并禁用 |
| 表单验证 | 数字输入框应限制范围和步长 |

### 8.4 后端注意事项

| 注意点 | 说明 |
|--------|------|
| Pydantic 验证 | 使用 Pydantic 模型自动验证字段范围和类型 |
| 日志记录 | 配置变更应记录日志便于审计 |
| 事务处理 | 多表更新时使用事务保证一致性 |
| 异常处理 | 区分 `ConfigValidationError` 和其他异常 |

## 10. 常见问题

### Q1: 修改配置后页面还是旧的怎么办？

**A**: 确保：
1. 前端保存成功后重新拉取配置
2. 或者刷新页面（ImportExportTab 导入后会自动刷新）

### Q2: 导入配置失败，提示校验错误？

**A**: 检查 YAML 文件：
1. 格式是否正确（缩进、冒号等）
2. 字段值是否在合法范围内
3. 权重总和是否为 1.0

### Q3: 导出的 YAML 文件中 API Key 为什么是空的？

**A**: 这是安全设计。API Key 属于敏感信息，导出时必须置空。导入时需手动填写。

### Q4: 多个 Tab 同时保存会冲突吗？

**A**: 每个 Tab 调用独立接口，不会冲突。但建议用户逐个 Tab 保存，避免配置不一致。

### Q5: 配置更新后需要重启服务吗？

**A**: 不需要。配置支持热更新，更新后立即生效。
