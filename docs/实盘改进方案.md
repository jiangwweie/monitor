## 改进一：修复仓位计算漏洞

### 问题分析

**问题 1**: 未考虑已有持仓的名义价值

```python
# 当前代码 (domain/risk/sizer.py:48)
investment_amount = account.total_wallet_balance / (4 - account.current_positions_count)
```

**问题场景**：
- 假设总资金 10000 USDT，已有 3 个持仓
- 当前逻辑：可用资金 = 10000 / (4-3) = 10000 USDT
- 实际情况：3 个持仓可能已占用 7500 USDT，实际可用仅 2500 USDT
- 结果：仓位计算严重超配

**问题 2**: 安全垫系数不足

```python
# 当前代码 (domain/risk/sizer.py:76)
theoretical_leverage = (notional_value / investment_amount) * 1.05
```

**风险**：
- 1.05 安全垫在极端行情下可能不足
- 维持保证金要求可能在波动中飙升至 10%+
- 存在未触达止损就被强平的风险

### 实施方案

**修改文件**: `domain/risk/sizer.py`

**核心变更**:

```python
class PositionSizer:
    # 安全垫系数 (v2.0: 从 1.05 提升到 1.15)
    SAFETY_CUSHION = 1.15

    # 最大持仓占用比例 (预留现金应对追加保证金)
    MAX_POSITION_USAGE = 0.85

    def calculate(self, signal, account, risk_pct, max_leverage):
        # ... 前置检查保持不变 ...

        # 【修复 1】精确计算可用资金
        simple_allocation = account.total_wallet_balance / (4 - account.current_positions_count)

        # 计算已有持仓的总占用
        existing_position_value = sum(
            pos.get('position_value', 0) for pos in account.positions
        ) if account.positions else 0

        # 可用资金 = 总资金 × 最大使用比例 - 已占用
        precise_allocation = (
            account.total_wallet_balance * self.MAX_POSITION_USAGE - existing_position_value
        ) / (4 - account.current_positions_count)

        # 取两者较小值（保守原则）
        investment_amount = min(simple_allocation, max(0, precise_allocation))

        if investment_amount <= 0:
            raise RiskLimitExceeded("精确计算后无可用资金")

        # ... 风险计算保持不变 ...

        # 【修复 2】使用提升后的安全垫
        theoretical_leverage = (notional_value / investment_amount) * self.SAFETY_CUSHION

        # ... 后续逻辑保持不变 ...
```

## 改进二：添加全局风控控制器

### 需求说明

当前系统仅有**单笔风险控制**（通过 `risk_pct` 限制单笔损失），缺失**累计风险监控**：

| 缺失功能 | 风险 |
|----------|------|
| ❌ 每日最大亏损限制 | 可能一天内亏光月利润 |
| ❌ 每周最大亏损限制 | 无周期性保护 |
| ❌ 连续止损熔断 | 连亏时继续开仓，损失扩大 |
| ❌ 最大回撤保护 | 极端情况下可能穿仓 |

### 实施方案

**新增文件**: `domain/risk/risk_controller.py`

**核心功能**:

class RiskState(str, Enum):

@dataclass
class RiskControllerConfig:
    """全局风控配置"""
    daily_loss_limit: float = 0.05       # 5% 日亏损限制
    weekly_loss_limit: float = 0.15      # 15% 周亏损限制
    consecutive_loss_limit: int = 3      # 连亏 3 单触发熔断
    cooldown_hours: int = 24             # 冷却 24 小时
    max_drawdown: float = 0.20           # 20% 最大回撤
    single_risk_limit: float = 0.02      # 2% 单笔风险（与 sizer 配合）

@dataclass
class TradeRecord:
    """交易记录"""
    symbol: str
    direction: str
    entry_price: float
    exit_price: float
    pnl: float                   # 实现盈亏 (USDT)
    pnl_pct: float               # 盈亏比例
    entry_time: int
    exit_time: int
    is_loss: bool

class RiskController:
    """
    全局风控控制器

    职责:
    1. 追踪每日/每周盈亏
    2. 统计连续亏损次数

    4. 触发熔断时禁止新开仓
    """

        self.config = config or RiskControllerConfig()
        self.state = RiskState.NORMAL

        self.peak_balance: float = 0.0
        self.initial_balance: float = 0.0
        self.cooldown_until: Optional[int] = None

        """初始化风控器"""
        self.initial_balance = initial_balance

        """更新峰值余额（用于回撤计算）"""
        if current_balance > self.peak_balance:
            self.peak_balance = current_balance

    def record_trade(self, trade: TradeRecord):
        """记录交易，用于风控统计"""
        self.trade_history.append(trade)
        if trade.is_loss:
            self._check_consecutive_losses()

    def check_can_open_position(self, current_balance: float) -> tuple[bool, str]:
        """
        检查是否允许开新仓

        :return: (是否允许，原因)
        """

        return {
            "state": self.state.value,

## 改进三：添加执行模块（可选）

### 需求说明

当前系统为 **Zero Execution** 架构（`auto_order_status` 硬编码为 `OFF`）。如需自动下单，需添加独立执行模块。

**设计原则**:
1. 与策略模块完全解耦，通过事件驱动通信
2. 所有网络请求带重试和熔断
3. 订单状态可追踪、可恢复
4. 作为可选插件，默认不启用

### 实施方案

**目录结构**:
```
domain/execution/
  __init__.py
  order_executor.py      # 订单执行器
  order_manager.py       # 订单状态管理
  stop_loss_monitor.py   # 止损监控器
```
