# 风控模块详细设计文档

## 1. 概述

风控模块是本监测系统的核心安全保障，负责计算安全的开仓仓位、杠杆倍数和风险金额，确保每笔交易的风险在可控范围内。

**设计原则**：
- **零执行**：仅提供计算建议，绝不执行实际订单
- **精度优先**：使用 `Decimal` 类型保证金融计算精度
- **多层防护**：前置拦截 + 仓位计算 + 杠杆熔断 + 组合风险聚合
- **纯领域逻辑**：无网络请求、无数据库操作、无副作用

**核心文件**：
- `domain/risk/sizer.py` - 仓位计算器
- `domain/risk/portfolio_risk.py` - 投资组合风险聚合服务

## 3. PositionSizer - 仓位计算器

### 3.1 核心方法签名

```python
def calculate(
    self,
    signal: Signal,
    account: AccountBalance,
    risk_config: RiskConfig
) -> PositionSizing:
    """
    根据当前账户快照和风控配置对有效信号进行杠杆和算仓建议计算。

    :param signal: 策略引擎生成的有效信号快照
    :param account: 从交易所实时只读获取的账户余额与持仓笔数
    :param risk_config: 风控配置参数
    :return: 推算出的包含杠杆及数量的 PositionSizing 实体
    :raises RiskLimitExceeded: 在前置拦截条件不满足时抛出异常
    """
```

### 3.2 参数有效性校验

```python
# 校验风险比例是否在合理范围内 (0, 10%]
if not (0 < risk_config.risk_pct <= 0.1):
    raise RiskLimitExceeded(
        "单笔风险比例必须在 (0, 10%] 范围内",
        error_code="INVALID_RISK_PCT"
    )

# 校验杠杆倍数是否在合理范围内 [1, 125]
if not (1 <= risk_config.max_leverage <= 125):
    raise RiskLimitExceeded(
        "杠杆倍数必须在 [1, 125] 范围内",
        error_code="INVALID_LEVERAGE"
    )
```

### 3.3 前置拦截规则

#### 规则 1：持仓笔数上限拦截

```python
if account.current_positions_count >= risk_config.max_positions:
    raise RiskLimitExceeded(
        f"当前持仓笔数已达上限 ({account.current_positions_count}>={risk_config.max_positions})",
        error_code="POSITION_LIMIT_EXCEEDED"
    )
```

**设计意图**：防止过度分散，保持持仓集中度，便于管理。

#### 规则 2：账户余额检查

```python
if account.total_wallet_balance <= 0 or account.available_balance <= 0:
    raise RiskLimitExceeded(
        "账户余额不足或已穿仓",
        error_code="INSUFFICIENT_BALANCE"
    )
```

**设计意图**：防止零除错误和穿仓状态下开仓。

### 3.4 本金分配计算

#### 固定比例分配策略

```python
# 使用 Decimal 保证精度
wallet_balance = Decimal(str(account.total_wallet_balance))
available_balance = Decimal(str(account.available_balance))

# 根据 max_positions 平均分配，每笔最多占用 1/max_positions 资金
max_position_ratio = Decimal("1") / Decimal(str(risk_config.max_positions))
max_investment = wallet_balance * max_position_ratio

# 实际投资不能超过可用余额
investment_amount = min(max_investment, available_balance)
```

**示例**：
- 账户总额：$10,000
- `max_positions = 4`
- 每笔最大投资：$10,000 / 4 = $2,500

### 3.5 风险及止损测算

#### 允许风险金额计算

```python
risk_pct = Decimal(str(risk_config.risk_pct))
original_risk_amount = wallet_balance * risk_pct
```

**示例**：
- 账户总额：$10,000
- `risk_pct = 0.02` (2%)
- 允许风险金额：$10,000 × 0.02 = $200

#### 止损距离校验

```python
sl_distance_pct = Decimal(str(signal.sl_distance_pct))

if signal.sl_distance_pct <= 0:
    raise RiskLimitExceeded(
        "信号自带的止损百分比距离无效或小于零",
        error_code="INVALID_SL_DISTANCE"
    )
```

### 3.6 仓位与杠杆计算

#### 名义价值 (Notional Value) 计算

```python
# 理论应开名义价值 = 单笔风险金额 / 止损距离比例
notional_value = original_risk_amount / sl_distance_pct
```

**示例**：
- 允许风险金额：$200
- 止损距离：3.5%
- 名义价值：$200 / 0.035 = $5,714.29

#### 仓位价值比例校验

```python
MAX_POSITION_VALUE_RATIO = Decimal("3.0")  # 名义价值不超过账户总额的 3 倍
max_allowed_notional = wallet_balance * MAX_POSITION_VALUE_RATIO

if notional_value > max_allowed_notional:
    raise RiskLimitExceeded(
        f"仓位价值比例超限",
        error_code="POSITION_VALUE_RATIO_EXCEEDED"
    )
```

**设计意图**：防止过度杠杆化，即使在高杠杆下名义价值也不应超过账户总额的 3 倍。

#### 开仓数量计算

```python
entry_price = Decimal(str(signal.entry_price))
suggested_quantity = notional_value / entry_price
```

**示例**：
- 名义价值：$5,714.29
- 入场价格：$3,500 (ETH 价格)
- 开仓数量：$5,714.29 / $3,500 = 1.63 ETH

#### 理论杠杆计算

```python
# 安全垫系数 1.15：预留 15% 的安全边际
safety_cushion = Decimal("1.15")
theoretical_leverage = notional_value / (investment_amount * safety_cushion)
```

**为什么需要安全垫？**
- 防止维持保证金不足导致强平
- 防止费率磨损导致止损未触发就爆仓
- 预留缓冲空间应对滑点和价格跳动

**示例**：
- 名义价值：$5,714.29
- 可用投资本：$2,500
- 安全垫后本金：$2,500 × 1.15 = $2,875
- 理论杠杆：$5,714.29 / $2,875 = 1.99x

### 3.7 杠杆熔断自适应

#### 熔断逻辑

```python
max_leverage = Decimal(str(risk_config.max_leverage))

suggested_leverage = theoretical_leverage
leverage_capped = False
actual_risk_amount = original_risk_amount

# 当理论杠杆超过系统上限时，强制压制
if suggested_leverage > max_leverage:
    leverage_capped = True
    suggested_leverage = max_leverage

    # 压缩名义价值（不再除以安全垫，因为杠杆已封顶）
    capped_notional_value = investment_amount * max_leverage

    # 重新计算开仓数量
    suggested_quantity = capped_notional_value / entry_price

    # 实际风险额由于杠杆封顶而降低
    actual_risk_amount = capped_notional_value * sl_distance_pct
```

**示例场景**：

假设止损距离非常小（如 0.5%），理论杠杆会非常高：

```
原始计算：
- 止损距离：0.5%
- 允许风险：$200
- 名义价值：$200 / 0.005 = $40,000
- 理论杠杆：$40,000 / ($2,500 × 1.15) = 13.9x

触发熔断 (max_leverage=10x)：
- 实际杠杆：10x（封顶）
- 压缩后名义价值：$2,500 × 10 = $25,000
- 实际开仓数量：$25,000 / $3,500 = 7.14 ETH
- 实际风险额：$25,000 × 0.005 = $125（从 $200 降至 $125）
```

### 3.8 输出结果

```python
@dataclass
class PositionSizing:
    signal: Signal
    suggested_leverage: float    # 建议杠杆（受限于 max_leverage）
    suggested_quantity: float    # 建议开仓数量
    investment_amount: float     # 分配的本金
    risk_amount: float           # 允许承担的固定风险额
    actual_risk_amount: float    # 实际风险额（杠杆熔断后可能降低）
    leverage_capped: bool        # 是否触发杠杆熔断
```

## 4. PortfolioRiskService - 投资组合风险聚合

### 4.1 核心职责

1. 聚合所有持仓的风险指标
2. 检查组合总风险是否超限
3. 提供风险透明度，帮助理解整体敞口

### 4.2 风险指标实体

```python
@dataclass
class PortfolioRiskMetrics:
    total_risk_amount: float      # 所有持仓的总风险额
    total_risk_pct: float         # 总风险占账户比例
    max_single_loss_pct: float    # 最大单笔损失比例
    position_count: int           # 持仓数量
```

### 4.3 组合风险计算

```python
def calculate_portfolio_risk(
    self,
    positions: List[Position],
    total_wallet_balance: float
) -> PortfolioRiskMetrics:
    # 空持仓或零余额时返回零值指标
    if not positions or total_wallet_balance <= 0:
        return PortfolioRiskMetrics(
            total_risk_amount=0.0,
            total_risk_pct=0.0,
            max_single_loss_pct=0.0,
            position_count=0
        )

    # 聚合所有持仓的风险额
    total_risk = sum(
        getattr(pos, 'risk_amount', 0.0) for pos in positions
    )
    total_risk_pct = total_risk / total_wallet_balance

    # 计算最大单笔风险
    max_single_risk = max(
        (getattr(pos, 'risk_amount', 0.0) for pos in positions),
        default=0.0
    )
    max_single_loss_pct = max_single_risk / total_wallet_balance

    return PortfolioRiskMetrics(
        total_risk_amount=total_risk,
        total_risk_pct=total_risk_pct,
        max_single_loss_pct=max_single_loss_pct,
        position_count=len(positions)
    )
```

### 4.4 组合风险上限检查

```python
def check_portfolio_limit(
    self,
    metrics: PortfolioRiskMetrics,
    max_portfolio_risk_pct: float = 0.08  # 默认 8%
) -> bool:
    """检查投资组合风险是否超限"""
    return metrics.total_risk_pct <= max_portfolio_risk_pct
```

**示例**：
- 账户总额：$10,000
- 当前持仓风险额：$150 + $200 + $180 = $530
- 总风险比例：$530 / $10,000 = 5.3%
- 上限：8%
- 结果：`True` (未超限，可以继续开仓)

## 5. 风控配置参数

### 5.1 RiskConfig 实体

```python
@dataclass
class RiskConfig:
    risk_pct: float           # 单笔最大风险百分比，例如 0.02 (2%)
    max_sl_dist: float        # 天地针熔断最大止损距离，例如 0.035 (3.5%)
    max_leverage: float       # 杠杆熔断上限，例如 20.0
    max_positions: int = 4    # 最大持仓数量上限
```

### 5.2 参数边界与默认值

| 参数 | 默认值 | 有效范围 | 说明 |
|------|--------|----------|------|
| `risk_pct` | 0.02 (2%) | (0, 0.1] | 单笔交易允许损失占账户总额的比例 |
| `max_sl_dist` | 0.035 (3.5%) | (0, 1.0) | 止损距离上限，防止"天地针"形态止损过大 |
| `max_leverage` | 20.0x | [1, 125] | 系统允许的最大杠杆倍数 |
| `max_positions` | 4 | [1, ∞) | 同时持有的最大仓位数量 |

### 5.3 配置影响分析

#### 保守配置示例
```python
RiskConfig(
    risk_pct=0.01,       # 1% 风险，非常保守
    max_sl_dist=0.02,    # 2% 止损上限
    max_leverage=5.0,    # 5 倍杠杆
    max_positions=3      # 最多 3 个仓位
)
```
**特点**：低风险、低回报、高安全性

#### 激进配置示例
```python
RiskConfig(
    risk_pct=0.05,       # 5% 风险，较激进
    max_sl_dist=0.08,    # 8% 止损上限
    max_leverage=50.0,   # 50 倍杠杆
    max_positions=8      # 最多 8 个仓位
)
```
**特点**：高风险、高回报、需严格监控

#### 推荐平衡配置
```python
RiskConfig(
    risk_pct=0.02,       # 2% 风险，平衡
    max_sl_dist=0.035,   # 3.5% 止损上限
    max_leverage=20.0,   # 20 倍杠杆
    max_positions=4      # 最多 4 个仓位
)
```
**特点**：风险可控、回报合理、适合大多数交易者

## 6. 异常处理

### 6.1 RiskLimitExceeded 异常

```python
@dataclass
class RiskLimitExceeded(Exception):
    message: str           # 错误消息
    error_code: str        # 错误代码
    context: dict          # 错误上下文信息
```

### 6.2 错误代码枚举

| 错误代码 | 触发条件 | 处理建议 |
|----------|----------|----------|
| `INVALID_RISK_PCT` | risk_pct 不在 (0, 10%] 范围 | 检查配置合法性 |
| `INVALID_LEVERAGE` | leverage 不在 [1, 125] 范围 | 检查杠杆配置 |
| `POSITION_LIMIT_EXCEEDED` | 当前持仓数 >= max_positions | 等待平仓或手动平仓 |
| `INSUFFICIENT_BALANCE` | 账户余额 <= 0 | 充值或等待出金到账 |
| `INVALID_SL_DISTANCE` | signal.sl_distance_pct <= 0 | 检查信号计算逻辑 |
| `POSITION_VALUE_RATIO_EXCEEDED` | 名义价值 > 账户总额 × 3 | 降低风险比例或增加本金 |

## 7. 计算示例

### 7.1 完整计算流程示例

**场景**：ETHUSDT 做多信号

**输入数据**：
```python

signal = Signal(
    symbol="ETHUSDT",
    direction="LONG",
    entry_price=3500.0,
    stop_loss=3380.0,       # 影线低点

)

# 风控配置

    risk_pct=0.02,          # 2%
    max_sl_dist=0.035,      # 3.5%
    max_leverage=20.0,
    max_positions=4
)
```

**计算过程**：

1. **参数校验**：通过
   - risk_pct = 0.02 ∈ (0, 0.1] ✓
   - max_leverage = 20.0 ∈ [1, 125] ✓

2. **前置拦截**：通过
   - current_positions_count = 2 < max_positions = 4 ✓
   - total_wallet_balance = $10,000 > 0 ✓

3. **本金分配**
   ```python
   max_position_ratio = 1 / 4 = 0.25
   max_investment = $10,000 × 0.25 = $2,500
   investment_amount = min($2,500, $8,500) = $2,500
   ```

4. **风险测算**
   ```python
   original_risk_amount = $10,000 × 0.02 = $200
   ```

5. **名义价值计算**
   ```python
   notional_value = $200 / 0.0343 = $5,830.90
   ```

6. **仓位价值校验**
   ```python
   max_allowed_notional = $10,000 × 3.0 = $30,000
   $5,830.90 < $30,000 ✓
   ```

7. **开仓数量计算**
   ```python
   suggested_quantity = $5,830.90 / $3,500 = 1.666 ETH
   ```

8. **理论杠杆计算**
   ```python
   safety_cushion = 1.15
   theoretical_leverage = $5,830.90 / ($2,500 × 1.15) = 2.03x
   ```

9. **杠杆熔断检查**
   ```python
   2.03x < max_leverage = 20.0x ✓
   未触发熔断
   ```

**输出结果**：
```python
PositionSizing(
    signal=signal,
    suggested_leverage=2.03,
    suggested_quantity=1.666,
    investment_amount=2500.0,
    risk_amount=200.0,
    actual_risk_amount=200.0,
    leverage_capped=False
)
```

## 8. 在引擎中的使用

### 8.1 实时监控流程

```python
# monitor_engine.py 中
async def start(self):
    async for current_bar in self.feed.subscribe_klines(...):
        # 1. 策略引擎生成信号
        signal = self.strategy.evaluate(...)

        if not signal:
            continue

        # 2. 读取账户余额
        account_balance = await self.account_reader.fetch_account_balance()

        # 3. 检查投资组合总风险
        portfolio_metrics = self.portfolio_risk_service.calculate_portfolio_risk(
            positions=account_balance.positions,
            total_wallet_balance=account_balance.total_wallet_balance
        )

        if not self.portfolio_risk_service.check_portfolio_limit(
            portfolio_metrics, self.max_portfolio_risk_pct
        ):
            logger.warning(f"投资组合总风险超限")
            continue

        # 4. 风控算仓
        try:
            sizing = self.risk_sizer.calculate(
                signal=signal,
                account=account_balance,
                risk_config=self.risk_config
            )
        except RiskLimitExceeded as e:
            logger.warning(f"风控拦截：{e}")
            continue

        # 5. 入库 + 推送
        await self.repo.save_signal(signal)
        await self.repo.save_position_sizing(sizing)
```

## 9. 安全边界分析

### 9.1 为什么需要安全垫？

```python
safety_cushion = Decimal("1.15")  # 15% 安全边际
```

**原因**：
1. **维持保证金**：开仓后需要额外保证金维持仓位
2. **强平缓冲**：防止价格小幅波动就触发强平
3. **费率磨损**：资金费率长期积累可能侵蚀本金
4. **滑点容忍**：极端行情下实际成交价可能偏离预期

**示例**：
```
无安全垫：
- 本金：$2,500
- 名义价值：$5,000
- 实际杠杆：2x
- 维持保证金率：0.5%
- 强平价格：接近入场价，极易触发

有安全垫：
- 本金：$2,500 × 1.15 = $2,875
- 名义价值：$5,000
- 实际杠杆：1.74x
- 维持保证金更充足
- 强平价格：远离入场价，更安全
```

### 9.2 为什么限制 max_positions？

**分散风险的边界**：

| max_positions | 单笔风险 | 总风险 | 特点 |
|---------------|----------|--------|------|
| 1 | 2% | 2% | 过度集中，风险过高 |
| 4 | 2% | 8% | 适度分散，推荐 |
| 10 | 2% | 20% | 过度分散，管理困难 |

**设计意图**：
- 防止过度分散导致单仓位影响力过低
- 保持适度集中，便于跟踪管理
- 限制总风险敞口

### 9.3 为什么使用 Decimal？

**浮点数精度问题示例**：
```python
# 使用 float
>>> 0.1 + 0.2
0.30000000000000004

# 使用 Decimal
>>> from decimal import Decimal
>>> Decimal("0.1") + Decimal("0.2")
Decimal('0.3')
```

**金融计算必须精确**：
- 风险金额计算涉及资金安全
- 杠杆计算误差可能导致爆仓
- 开仓数量误差可能导致资金利用率低下

## 11. 总结

风控模块是系统的核心安全保障，关键特性：

- ✅ **多层防护**：前置拦截 + 仓位计算 + 杠杆熔断 + 组合风险
- ✅ **精度优先**：使用 `Decimal` 保证金融计算零误差
- ✅ **安全垫设计**：15% 安全边际防止强平
- ✅ **动态适应**：杠杆熔断根据理论风险自适应调整
- ✅ **组合视角**：聚合所有持仓风险，防止整体失控
- ✅ **零执行**：严格遵守只计算不交易的底线

