# 技能：动态配置热重载与 Web 安全

## 核心目标
系统必须支持在不重启进程的情况下，通过 Web API 修改策略风控参数，并在下一个逻辑循环中立即生效。

## 严格约束规则
1. **严格的入参校验 (Pydantic Defenses)**：
   - Web 层的 API 接口必须使用 `pydantic` 的 `Field(ge=x, le=y)` 定义数据验证范围。
   - 绝不相信前端传来的数据，必须在到达数据库前拦截所有非法的异常风控参数（如杠杆 > 125，风险比例 > 0.05）。
2. **状态分离解耦**：
   - Web 层 (`web/api.py`) 和核心引擎 (`application/monitor_engine.py`) 之间**禁止**通过全局变量或内存单例直接通信。
   - Web 层只负责把配置写入 SQLite 数据库 (`UPDATE ...`)。
   - 引擎层在每次处理新的 K 线之前，调用 `IRepository` 接口从 SQLite 读取最新配置，实现天然的热加载。
3. **数据库异步化**：
   - 所有的数据库存取操作必须使用 `aiosqlite` 异步库，严禁使用同步的 `sqlite3` 阻塞主事件循环。
